Технічна архітектура IP-KVM Control Plane

  Система побудована як централізований вузол керування для віддалених IP-KVM пристроїв на базі Raspberry Pi.
   - Backend (FastAPI): Виконує роль сигнального сервера WebRTC, безпечного WebSocket-проксі для передачі команд HID та менеджера станів вузлів.
   - Frontend (Vue.js 3 + Vuetify): Забезпечує інтерфейс для моніторингу, відтворення потокового відео з низькою затримкою та захоплення вводу користувача.
   - VPN/Tunnel (Cloudflare): Використовується для безпечного з'єднання бекенду з віддаленими вузлами без необхідності відкриття портів.

  ---

  Детальна структура каталогів та опис файлів

  📂 backend/
  ├─── main.py — Точка входу FastAPI: конфігурує додаток, підключає роутери та запускає фоновий сервіс моніторингу вузлів.
  ├─── alembic.ini — Конфігураційний файл інструменту Alembic для керування міграціями бази даних.
  ├─── 📂 alembic/ — Директорія з логікою міграцій (env.py керує підключенням Alembic до SQLAlchemy).
  │    └─── 📂 versions/ — Файли міграцій (.py), що описують зміни структури БД (додавання полів для скріншотів, URL тунелів тощо).
  ├─── 📂 api/ — Модулі, що визначають REST та WebSocket ендпоінти.
  │    ├─── auth.py — Обробка логіну, оновлення JWT-токенів та виходу з системи.
  │    ├─── kvm_nodes.py — CRUD операції для керування KVM-вузлами та отримання їхнього статусу.
  │    ├─── signaling.py — Реалізація WHEP-протоколу: проксіює SDP-оффери від клієнта до MediaMTX на Raspberry Pi.
  │    ├─── ws_proxy.py — Ендпоінт для WebSocket-з'єднання, що ініціює безпечний проксі-міст до вузла.
  │    └─── __init__.py — Експортує роутери для підключення в основний додаток.
  ├─── 📂 core/ — Ядро системи: налаштування та безпека.
  │    ├─── config.py — Централізоване керування налаштуваннями через Pydantic-settings (зчитування .env).
  │    ├─── dependencies.py — Фабрики залежностей FastAPI для перевірки JWT та прав доступу користувача до конкретних вузлів.
  │    ├─── security.py — Утиліти для хешування паролів (bcrypt) та генерації/валідації JWT токенів.
  │    └─── token_schemas.py — Pydantic-моделі для структури корисного навантаження (payload) токенів.
  ├─── 📂 db/ — Робота з базою даних.
  │    ├─── init_db.py — Скріпт для ініціалізації БД та створення першого суперкористувача (admin).
  │    ├─── session.py — Налаштування асинхронного двигуна SQLAlchemy та фабрики сесій.
  ├─── 📂 models/ — Опис таблиць БД (SQLAlchemy/SQLModel).
  │    ├─── kvm_node.py — Модель KvmNode: зберігає IP, URL тунелю, статус та інформацію про залізо вузла.
  │    ├─── user.py — Модель User: облікові дані, прапорці активності та прав суперкористувача.
  │    ├─── user_node_permission.py — Модель зв'язку: визначає права конкретного користувача на перегляд/керування вузлом.
  ├─── 📂 schemas/ — Pydantic-моделі для валідації вхідних/вихідних даних API.
  │    ├─── auth.py — Схеми для запитів на логін та відповідей з токенами.
  │    ├─── kvm_node.py — Схеми для створення, оновлення та відображення даних вузла.
  │    └─── signaling.py — Схеми для обміну SDP та ICE-кандидатами під час WebRTC-сигналінгу.
  ├─── 📂 services/ — Бізнес-логіка, відокремлена від API.
  │    ├─── node_health.py — Фоновий сервіс, що періодично опитує вузли (ping/options) та оновлює їх статус у БД.
  │    ├─── node_manager.py — Інкапсульована логіка CRUD для роботи з об'єктами вузлів у базі.
  │    ├─── node_url.py — Допоміжні функції для формування коректних URL (HTTP/WS) до вузлів залежно від наявності тунелю.
  │    └─── ws_proxy.py — Основна логіка асинхронного WebSocket-проксі: двостороннє пересилання повідомлень між клієнтом та вузлом.

  📂 frontend/
  ├─── vite.config.ts — Конфігурація збирача Vite (проксі, плагіни, аліаси).
  ├─── 📂 src/ — Основний вихідний код фронтенду.
  │    ├─── App.vue — Кореневий компонент додатку, що містить основний layout.
  │    ├─── main.ts — Точка входу: ініціалізація Vue, Pinia, Vuetify та роутера.
  │    ├─── 📂 components/ — Повторно використовувані UI компоненти.
  │    │    ├─── AddNodeDialog.vue — Діалогове вікно для реєстрації нового KVM-вузла суперкористувачем.
  │    │    ├─── NodeCard.vue — Картка вузла для дашборду (відображає статус, назву та прев'ю).
  │    │    ├─── WebRTCCaptureOverlay.vue — Оверлей для захоплення фокусу миші та клавіатури над відео-плеєром.
  │    │    ├─── WebRTCPlayer.vue — Складний компонент, що поєднує відео-елемент, логіку стрімінгу та передачу вводу.
  │    │    └─── WebRTCStatusOverlay.vue — Візуалізація станів підключення (Loading, Error, Connected) поверх відео.
  │    ├─── 📂 composables/ — Логіка у стилі Vue Composition API.
  │    │    ├─── useHID.ts — Керує WebSocket-з'єднанням для передачі команд миші/клавіатури до бекенду.
  │    │    ├─── useKeyboardInput.ts — Обробник подій клавіатури: мапінг клавіш у формат HID-репортів.
  │    │    ├─── useMouseInput.ts — Обробник подій миші: обчислення відносного руху та стану кнопок.
  │    │    ├─── usePlayerInput.ts — Координатор, що активує захоплення вводу при кліку на плеєр.
  │    │    └─── useWebRTC.ts — Логіка встановлення WebRTC-з'єднання, отримання стріму та створення скріншотів.
  │    ├─── 📂 router/ — Конфігурація маршрутів (Login, Dashboard, Stream View).
  │    ├─── 📂 stores/ — Керування станом (Pinia).
  │    │    └─── auth.ts — Зберігання JWT-токенів у LocalStorage та перевірка статусу авторизації.
  │    ├─── 📂 utils/ — Допоміжні функції.
  │    │    └─── hid.ts — Мапінг JavaScript KeyboardEvent.code у стандартні USB HID keycodes.
  │    └─── 📂 views/ — Основні сторінки (Page Components).
  │         ├─── DashboardView.vue — Список доступних вузлів з можливістю пошуку та додавання.
  │         ├─── LoginView.vue — Форма авторизації користувача.
  │         └─── NodeStreamView.vue — Сторінка віддаленого керування: відео-потік та панель інструментів.
